Few-shot
예시를 몇 개 보여주기
특정 형식이나 패턴을 모델에게 제시
도할 때 유용
정해진 포맷의 답변을 유 복잡한 논리 추론 문제에서는
예시만으로 한계가 있음

Chain of Thought(CoT)
풀이 과정을 나열함
모델의 논리적 추론 능 력을 활성화
정답률을 높일 수 있음
복잡한 수학, 상식 추론의 중간 단계가 틀리면 최종 답도
틀리는 '오류 전파' 발생

Zero-shot CoT
"차례 대로 풀자" 라는 특정 문구로 단계 생성
예시(Few-shot) 없이 도 추론 능력 유도
예시를 직접 작성할 필요 가 없어 매우 간편함
Few-shot CoT에 비해 논리 전개가 다소 불안정

Least-to-Most
문제를 하위 문제로 쪼개기
시보다 훨씬 복잡한 다단계 문제 해결
긴 추론 과정이 필요한 고난도 문제에 강함
여러 번의 단계적 질문이 필요하여 비용/시간 증가

Self-Consistency
여러 답을 생성하고 다수결 투표
답변의 신뢰성과 정확도 검증
모델의 불안정한 실수를 방지하고 정답의 확신도 높임
여러 번 생성해야 하므로 API 비용이 많이 발생함


Chains (체인): 여러 작업(LLM 호출, 데이 터 처리 등)을 순차적 으로 연결하는 시퀀스 를 편리하게 정의
Agents (에이전트): LLM을 추론 엔진으로 사용하 여 어떤 행동(Action)을 취 할지, 어떤 도구를 사용할 지 동적으로 결정
Prompts (프롬프트): LLM에 입력할 지시사항 을 포멧팅 & 관리
Retrieval (검색): 외부 데이터 (PDF, 웹사이트 등)를 로드하 고 검색하여 LLM에 제공할 수 있도록 벡터DB 혹은 API를 제 공 (RAG 구현 시 필수)
Memory (메모리): 대화 의 맥락을 유지하기 위 해 이전 상호작용 데이 터를 저장하고 관리
Models (모델): LLM(텍스트 생성) 및 Chat Models(대화 형 메시지 처리)에 대한 표준 인터페이스를 제공


LangChain은 거대 언어 모델을 사용하여 에이전트 애플리케이션을 쉽게 구축할 수 있도록 돕는 오픈 소스 프레임워크

LangChain LCEL Chain
구조 - 파이프(|) 연결로 선형 흐름에 최적화
조건 분기 - RunnableBranch로 가능하지만, 분기 후 재합류가 번거로움
순환/반복(Loop) - 기본적으로 불가 (LCEL 체인 밖에서 루프 로 감싸 처리)
코드 가독성 - 짧거나 선형적인 파이프라인은 간단하고 코드는 짧음
데이터(상태) 관리 - RunnablePassthrough()으로 dict를 수 동으로 누적 관리

LangGraph StateGraph
구조 - 노드•엣지 그래프 구조로 비선형 흐름 자유롭 게 표현
조건 분기 - 분기•합류•다중 경로 구현이 손쉽게 가능
순환/반복(Loop) - 그래프 안에 루프를 직접 표현 가능 (에이전트 의 "생각-행동-관찰" 반복에 적합)
코드 가독성 - 노드 함수를 분리 정의하므로 구조 파악이 쉬우 나 코드는 길어짐
데이터(상태) 관리 - 클래스를 명시적 선언하여 자료구조를 줄 수 있 으며 그래프 실행 중 자동 전달
